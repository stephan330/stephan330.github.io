Docker pour les débutants : Comprendre les conteneurs

Vous avez entendu parler de Docker partout, mais vous ne savez pas par où commencer ? Cet article vous guide à travers les concepts fondamentaux de la conteneurisation et vous montre comment créer vos premiers conteneurs.

🤔 Qu'est-ce que Docker ?

Docker est une plateforme de conteneurisation qui permet d'empaqueter une application avec toutes ses dépendances dans un « conteneur ». Ce conteneur peut ensuite être exécuté sur n'importe quelle machine disposant de Docker, sans se soucier de l'environnement.

Imaginez un carton de déménagement : vous mettez tout ce dont votre application a besoin (code, bibliothèques, configuration) dans ce carton. Peu importe où vous le déplacez, tout fonctionnera exactement de la même manière.

📦 Les concepts clés

Image Docker

Une image est un modèle en lecture seule qui contient les instructions pour créer un conteneur. C'est comme une recette de cuisine : elle décrit tous les ingrédients et les étapes de préparation.

Conteneur

Un conteneur est une instance exécutable d'une image. Si l'image est la recette, le conteneur est le plat cuisiné. Vous pouvez créer plusieurs conteneurs à partir de la même image.

Dockerfile

Le Dockerfile est un fichier texte qui contient les instructions pour construire une image Docker. Voici un exemple simple : 

# Utiliser une image PHP officielle
FROM php:8.2-apache

# Copier le code source
COPY . /var/www/html/

# Exposer le port 80
EXPOSE 80

# Démarrer Apache
CMD ["apache2-foreground"]

🚀 Premiers pas pratiques

1. Installer Docker

Sur Ubuntu/Debian, l'installation est simple :

# Mettre à jour les paquets
sudo apt update

# Installer Docker
sudo apt install docker.io

# Vérifier l'installation
docker --version

2. Votre premier conteneur

Lancez votre premier conteneur avec cette commande :

# Télécharger et exécuter un conteneur Nginx
docker run -d -p 8080:80 nginx

Ouvrez votre navigateur à http://localhost:8080 — vous verrez la page d'accueil de Nginx ! 🎉

3. Commandes essentielles

# Voir les conteneurs en cours d'exécution
docker ps

# Voir toutes les images téléchargées
docker images

# Arrêter un conteneur
docker stop [ID_CONTENEUR]

# Supprimer un conteneur
docker rm [ID_CONTENEUR]

# Construire une image à partir d'un Dockerfile
docker build -t mon-app .

💡 Astuce : Utilisez docker-compose pour gérer plusieurs conteneurs en même temps (par exemple : une app PHP + une base MySQL). C'est un outil indispensable pour les projets plus complexes !

🎯 Pourquoi utiliser Docker ?

  • Portabilité — « Ça marche sur ma machine » ne sera plus jamais un problème
  • Isolation — Chaque application est isolée dans son conteneur
  • Reproductibilité — L'environnement est identique partout
  • Rapidité — Les conteneurs démarrent en quelques secondes
  • CI/CD — Intégration parfaite dans les pipelines d'automatisation

📚 Pour aller plus loin

Docker est un outil vaste et passionnant. Pour approfondir vos connaissances, je vous recommande la documentation officielle de Docker et de pratiquer avec des projets personnels. C'est en forgeant qu'on devient forgeron ! 🔨
